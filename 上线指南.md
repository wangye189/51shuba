# 51书库 上线指南（零服务器零成本）

架构：**Next.js → Vercel（免费）** + **数据 → Turso 云库（免费）**。你不用买服务器、不用运维数据库。

> ⚠️ 注册账号这步只能你本人做（我无法代注册）。其余我都能配合。

---

## 第 1 步：注册 Turso，建数据库（约 5 分钟）

1. 打开 https://turso.tech ，用 GitHub/邮箱注册（免费层）
2. 安装 Turso CLI（Mac 终端）：
   ```bash
   curl -sSfL https://get.tur.so/install.sh | bash
   turso auth login
   ```
3. 建库 + 取凭据：
   ```bash
   turso db create book51
   turso db show book51 --url          # → 这是 TURSO_DATABASE_URL
   turso db tokens create book51       # → 这是 TURSO_AUTH_TOKEN
   ```
   把这两个值发我，或自己留好。

## 第 2 步：把本地数据推上 Turso（我帮你跑）

项目根执行（用上一步的两个值）：
```bash
TURSO_DATABASE_URL=libsql://book51-xxx.turso.io \
TURSO_AUTH_TOKEN=xxxxx \
node scripts/push-to-turso.mjs
```
看到「✓ 完成。Turso 现有 N 本书 / M 章」就成了。

## 第 3 步：代码推到 GitHub

```bash
cd /Users/mac/Claude/book51
git init && git add -A && git commit -m "51书库 MVP"
# 在 github.com 建个空仓库 book51（私有即可），然后：
git remote add origin git@github.com:你的账号/book51.git
git push -u origin main
```

## 第 4 步：注册 Vercel，部署（约 5 分钟）

1. 打开 https://vercel.com ，用 GitHub 登录（免费）
2. **Add New → Project**，选你的 `book51` 仓库，Import
3. 在 **Environment Variables** 里填：
   | 变量 | 值 |
   |---|---|
   | `TURSO_DATABASE_URL` | 第 1 步的 url |
   | `TURSO_AUTH_TOKEN` | 第 1 步的 token |
   | `NEXT_PUBLIC_SITE_URL` | 部署后 Vercel 给的域名，如 `https://book51.vercel.app` |
   | `ADMIN_KEY` | 你自己设个友链后台口令 |
4. **Deploy**。几分钟后给你一个 `xxx.vercel.app` 域名，**这就是上线地址**。
5. （可选）`NEXT_PUBLIC_ADSENSE_CLIENT` / `NEXT_PUBLIC_GA_ID` 等广告统计变量，过审后再填。

## 之后更新内容

采新书入本地库 → 重跑第 2 步（push-to-turso）→ Vercel 自动用新数据。
代码改动 → `git push` → Vercel 自动重新部署。
